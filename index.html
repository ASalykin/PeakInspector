PeakInspector

View on GitHub Download .zip Download .tar.gz

PeakInspector represents a simple standalone application for interactive analysis of peak properties in time-series data. PeakInspector allows to preprocess data before the analysis by baseline normalising, linear detrending (optionally) and smoothing data (by Savitzky–Golay filter). For the full list of a peak properties which could be currently analysed by PeakInspector please refer to this checklist.

Requirements

PeakInspector is written in Python 3.x, and generally platform-independent. The following packages are required:

  • PyQt4
  • numpy
  • scipy
  • matplotlib
It is recommended to install Anaconda package which already contains all required packages. For better experience it is recommended to work with PeakInspector on monitor with FullHD resolution (1920x1080).

Installation

It is possible to use PeakInspector by simple downloading the archive (on the top page), or clone this repository.

Working with PeakIspector

Explanation of every option on the graphical interface could be obtained by simple cursor-over the option and waiting for 1-2 seconds. Explanation tip will automatically pop-up.

  • Load data

    • Data file could be loaded by simple click on "Load file" button. File should be in text format, and contain 1 (time invariant, data indeces will be considered on time scale) or 2 columns of data (in case specifically recorded time). Delimiter between columns could be explicitly chosen. If data contain header, or any additional information at the beginning or end of the file one could skip this rows by specifying how many rows to skip. Data will be loaded and preprocessed automatically.

  • Analyse peaks

    • Dy default the option "Detrend data" is enabled. As well as some options for Savitzky–Golay filter are also specified. If not needed, user can disable "Detrend data" option, as well as change parameters for Savitzky–Golay filter. To see the changed graphical ouput user have to press the "Replot" button.

    The general procedure of peak analysis include clicking by the left mouse button near the peak local maximum (with enabled option "Enable peak choosing"). PeakInspector automatically search for the true local peak maximum (taking into account the option "Peak detection window", which specifies the "distance" in data points withing which the peak maximum will be searched from the initial user click). Along with that, PeakInspector will try to find where peak starts and ends, and calculate properties of the peak, which include:

    • Time of the peak maximum
    • Time of the peak start
    • Time of the peak end
    • Peak start value (ordinate)
    • Peak end vale (ordinate)
    • Absolute amplitude (A, according to the normalized baseline, which is 0)
    • Maximal amplitude among all peaks in current data file
    • Absolute amplitude relative to the maximal amplitude among all peaks in current data
    • Relative amplitude to the original baseline (absolute amplitude / original baseline value for current peak)
    • Maximal relative amplitude among all peaks in current data file
    • Relative amplitude relative to the maximal relative amplitude among all peaks in current data
    • Period (P, time between 2 nearest peak maxima)
    • Frequency (1/P, just value, opposite to period)
    • Halfdecay time
    • Halfdecay amplitude
    • Time to peak (from peak start to peak maximum)
    • Decay time (from peak maximum to peak end)
    • Full peak time (from peak start to peak end)
    • AUC (area under the curve, or peak area)
    • Frequency (Hz) of big peaks in current data file (with absolule amplitude >=0.66)
    • Frequency (Hz) of medium peaks in current data file (with absolule amplitude >=0.33 and <0.66)
    • Frequency (Hz) of small peaks in current data file (with absolule amplitude <0.33)

    User can interactively move the peak borders (start and end points of the peak) by simply drag the corresponding point by right mouse button, or explicitly point to the region where it shoud be by pressing CTRL+Right mouse button for the peak start, and ALT+Right mouse button for the peak end point.

  • Save ouput

    • All files analysed are considered as a batch (one treatment, one biological repetition, etc), and clicking on the button "Analyse and save file" save currenly analysed file into table inside PeakInspector, and add it to previously analysed files from the same batch. To save the all analysed peak from the batch as a table (for now the default output is Excel file) user have to click the "Export analysed files to Excel".
